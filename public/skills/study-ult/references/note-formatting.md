# Note Formatting Reference

## Required Callout Types

Use these callouts in EVERY topic note:

### Conceptual Callouts

```markdown
> [!KEY-CONCEPT]
> Core definition or fundamental principle.
> This is what the student MUST know.

> [!DEEP-INSIGHT]
> Understanding that goes beyond textbook.
> Separates top 1% students from others.

> [!INTUITION]
> Visual or mental model to understand concept.
> How experts think about this.
```

### Warning Callouts

```markdown
> [!COMMON-MISTAKE]
> Error: [Specific mistake]
> Why: [Reason it happens]
> Fix: [Correct approach]

> [!TRAP]
> Trap: [How trap is set]
> How to avoid: [Method]

> [!IMPORTANT]
> Critical point that cannot be missed.
> Frequently tested concept.
```

### JEE-Specific Callouts

```markdown
> [!JEE-INSIGHT]
> What JEE examiners look for.
> Common exam patterns.

> [!EXAM-PATTERN]
> How this topic appears in JEE.
> Frequency: High/Medium/Low

> [!SHORTCUT]
> Fast solving method.
> Time saved: ~X minutes
```

### Example Callouts

```markdown
> [!EXAMPLE]
> Application of concept.
> Follow the thinking process.

> [!TIP]
> Quick trick or memory aid.
> Useful for problem solving.
```

## Section Headers Required Per Note

Every topic note must have ALL of these sections:

```markdown
## Topic Overview
[1 paragraph - what this is]

## Core Theory  
[Detailed - this is where length matters]

## Intuitive Explanation
[How to THINK, not just what]

## Physical Meaning
[Real world interpretation]

## Key Formula(s)
$$[LaTeX formula]$$

## Important Observations
[Patterns students must recognize]

## Common Confusions
[Why students get confused]

## Common Mistakes
> [!COMMON-MISTAKE]
> [Specific error]

## Solved Examples
### Example 1: [Descriptive]
**Problem:** [Statement]
**Approach:** [How to think]  
**Solution:** [Steps with explanations]

> [!JEE-INSIGHT]
> [Exam relevance]

## JEE Insights
> [!EXAM-PATTERN]
> [How it appears in JEE]

## Memory Tricks
> [!INTUITION]
> [Visual/mnemonic aid]

## Related Concepts
- [[Topic 1]]
- [[Topic 2]]
- [[Formula Sheet]]
```

## Formatting Best Practices

1. **Use headings hierarchy**: # → ## → ### → #### 
2. **Add spacing**: Blank lines between sections
3. **Use tables**: For comparisons, formula summaries
4. **Use LaTeX**: For all mathematical expressions
5. **Be verbose**: Don't compress explanations
6. **Add examples**: Minimum 2 per topic
7. **Link everything**: Wikilinks to related topics
